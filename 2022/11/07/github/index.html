How & Why use github?

Posted on 2022-11-07 Edited on 2022-11-08

협업할 수 있는 능력을 가지자 / 부제: Github을 쓸줄 알아야지

Webpack과 commit을 연습중

Github이 무엇인가

서비스이다. GIT을 활용한 버전관리를 온라인상에서 저장할 수 있도록 해주는 서비스입니다. 서비스를 강조한 이유는 Github과 Git은 엄연히 다르다는 것을 기억해두기 위함입니다.

내가 그동안 사용한 Github

  1. git add . && git commit -m "Blah Blah" && git push

    와 github repository에 성공적으로 올라갔다. Github 써보았다.

  2. git add를 선택적으로 그리고 git commit -m 보다 git commit을 통한 커밋 저장이 왜 중요한가?

  • git은 버전관리 또는 형상관리를 위한 도구입니다. 하나의 서비스를 만드는 회사나, 팀이 Github으로 버전을 관리한다는 것은 나 이외에 다른 사람과 커밋을 공유한다는 뜻입니다. 그래서 정해진 커밋 컨벤션에 맞게 커밋을 작성하고 그 커밋은 동작하는 하나의 묶을으로 add하여 의미있는 파일들을 Repository에 push해야 합니다.
  • 아직 개발자로 경험해본적도 없는 제가 어떻게 이를 느꼈을까요? Conventional Commits 많은 개발자들은 표준화된 것을 좋아합니다. 그래서 널리 알려진 컨벤션들을 활용하는 회사도 많습니다. 오픈소스를 읽어도 git clone 뿐이였습니다. 하지만 컨벤션을 지킨 이슈와 풀리퀘스트 등을 관찰할 수 있게 되었습니다.
  • 대표적 예로 toss/slash 작성일(11-08) 기준으로 토스에서 오픈소스를 두어달 전에 오픈했습니다. 오픈소스에 이슈들과 풀리퀘스트를 읽고 이것이 BUG이고 FEAT 이런 컨벤션이 지켜져 올라가는 것을 보고 저 또한 일면식 없는 사람들의 메세지를 이해하고 읽을 수 있었습니다.

    toss/slash는 타입스크립트로 이루어진 패키지 모음입니다. 읽을 수 있지만 컨트리뷰트를 아직 못하는 것이 정말 아쉽습니다. 아직 ES6+와 Webpack에 집중하고 있습니다. 그래서 타입스크립트을 잘 알지못합니다.
